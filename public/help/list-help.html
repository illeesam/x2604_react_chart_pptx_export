보고서 목록 화면 구성

이 화면은 샘플 보고서 목록을 표 형태로 보여 주고, 같은 데이터로 미리보기·다운로드를 수행합니다.

  • 제목 영역: 「보고서 목록」과 안내 문구, 오른쪽에는 데이터 로드 상태(로딩 중 / 실패 / 완료) 배지가 표시됩니다.
  • 테이블
    • 번호: 행 식별용 순번입니다.
    • 보고서명: 데모용 제목입니다.
    • 담당부서 · 생성일 · 상태: 목록 메타 정보입니다. 상태는 색상 뱃지(최종·검토중·초안)로 구분됩니다.
    • 작업: 미리보기(모달), 형식 선택(드롭다운), 내려받기(선택 형식으로 파일 저장)입니다.
  • 하단 안내 박스: /api/pptData.json을 axios로 불러오는 방식과, 미리보기에서 렌더되는 페이지 구성에 대한 요약입니다.
앱 전체 상단 네비(ReportHub, 시스템 부제)는 모든 화면에 공통으로 표시됩니다.

주요 기능

  • 데이터 로드: 페이지가 열리면 GET /api/pptData.json(정적 파일)을 axios로 읽어 보고서 본문 데이터를 채웁니다. 실패 시 상단에 오류가 표시됩니다.
  • 미리보기: 로드된 JSON을 기준으로 표지와 차트 3페이지 · 데이터 4페이지(총 8개 탭)을 넘겨 보며 확인할 수 있습니다. 목록의 「이미지」 내려받기는 여기서 페이지별 이미지 저장을 안내합니다.
  • 내려받기(목록): 행마다 형식을 고른 뒤 「내려받기」로 전체 보고서를 한 번에 저장합니다. README(마크다운), PDF, PPT(캡처 기반 슬라이드), PPTX(네이티브 차트·표), HTML 등을 지원합니다.
  • 미리보기 내 추가 기능: 현재 페이지 단위·전체 일괄 저장(이미지, PDF, PPT, PPTX, HTML 등)을 더 세밀하게 선택할 수 있습니다.
목록의 보고서 행은 데모용이며, 본문 데이터는 동일한 pptData.json을 공유합니다.

버튼·조작별 안내

보고서 목록 화면과 미리보기 모달에서 쓰는 주요 조작을 표로 정리했습니다.

화면 기능 설명 사용된 라이브러리 비고
보고서 목록 도움말 제목 「보고서 목록」 오른쪽 버튼을 누르면 이 도움말 페이지를 새 브라우저 창으로 연다. React(클릭), window.open, 정적 HTML 팝업 차단 시 브라우저에서 허용 필요. 경로는 /help/list-help.html
미리보기 해당 행의 보고서를 로드된 JSON 기준으로 모달에서 넘겨 본다. React pptData.json 로드 전·실패 시 비활성
README 내려받기 드롭다운에서 README를 선택한 뒤 「내려받기」를 누르면 전체 보고서(표지~데이터 요약)를 마크다운(.md) 파일로 저장한다. 브라우저 Blob, 다운로드 링크, 문자열 직렬화 행마다 드롭다운 선택은 독립적. 데이터는 이미 메모리에 로드됨
이미지 내려받기 드롭다운에서 이미지를 선택한 뒤 「내려받기」를 누르면, 목록 화면에서는 파일 대신 미리보기에서 페이지별 이미지 저장을 안내하는 alert만 표시한다. React(alert) 실제 PNG 저장은 미리보기의 「이미지」 또는 「전체이미지」 사용
PDF 내려받기 드롭다운에서 PDF를 선택한 뒤 「내려받기」를 누르면 표지~데이터까지 페이지를 순서대로 렌더·캡처해 한 개의 PDF로 저장한다. react-dom createRoot, html2canvas, jspdf 처리 중 진행률 표시. 가로 A4 기준
PPT 내려받기 드롭다운에서 PPT를 선택한 뒤 「내려받기」를 누르면 각 페이지를 캡처해 이미지 슬라이드로 이어 붙인 .pptx 한 파일로 저장한다. react-dom createRoot, html2canvas, pptxgenjs 차트는 래스터 이미지로 들어감
PPTX 내려받기 드롭다운에서 PPTX를 선택한 뒤 「내려받기」를 누르면 전체 데이터로 네이티브 차트·표 중심의 여러 슬라이드 .pptx를 한 파일로 생성한다. pptxgenjs 캡처 파이프라인 없이 데이터 직접 반영
HTML 내려받기 드롭다운에서 HTML을 선택한 뒤 「내려받기」를 누르면 표지~데이터 페이지를 순서대로 캡처해 이미지를 임베드한 한 개의 HTML 파일로 저장한다. react-dom createRoot, html2canvas, Blob 처리 중 진행률 표시
미리보기 닫기 ✕ 미리보기 모달을 닫고 목록으로 돌아간다. React 전체 파일 생성 처리 중에는 비활성
README 현재 탭 화면이 아니라, 앱에 정의된 전체 보고서 데이터를 마크다운(.md)으로 저장한다. 브라우저 Blob, 다운로드 링크, 문자열 직렬화 파일명에 페이지 번호 등이 붙을 수 있음
이미지 현재 보이는 본문 영역만 PNG로 캡처해 저장한다. html2canvas scale 2 등 옵션은 코드 기준
PDF 본문 영역을 캡처해 가로 A4 PDF 한 파일로 만든다. 세로가 길면 페이지를 나눈다. html2canvas, jspdf 현재 탭 1페이지 분량
PPT 본문을 이미지 한 장으로 캡처해 슬라이드 1장짜리 .pptx로 저장한다. html2canvas, pptxgenjs 차트는 래스터 이미지로 들어감
PPTX 현재 페이지에 맞는 데이터만으로 PowerPoint에서 편집 가능한 차트·표 위주 슬라이드를 만든다. pptxgenjs 일부 차트 타입은 라이브러리 매핑·폴백 있음
HTML 캡처한 이미지를 data URL로 넣은 단일 HTML 파일로 저장한다. html2canvas, Blob 오프라인에서도 이미지가 보이도록 임베드
전체이미지 표지(0)~데이터(7)까지 순서대로 화면 밖에 렌더한 뒤 페이지마다 캡처하여 PNG 파일을 여러 개 저장한다. react-dom createRoot, html2canvas 진행률 UI 표시. 페이지당 파일 1개
전체PDF 표지(0)~데이터(7)까지 순서대로 화면 밖에 렌더한 뒤 캡처·병합하여 한 개의 PDF로 저장한다. react-dom createRoot, html2canvas, jspdf 진행률 UI 표시. 가로 A4 기준
전체PPT 표지(0)~데이터(7)까지 순서대로 화면 밖에 렌더한 뒤 각 페이지를 캡처해 슬라이드로 이어 붙인 한 개의 .pptx로 저장한다. react-dom createRoot, html2canvas, pptxgenjs 진행률 UI 표시. 이미지 슬라이드 방식
전체HTML 표지(0)~데이터(7)까지 순서대로 화면 밖에 렌더한 뒤 캡처한 이미지를 섹션별로 임베드한 한 개의 HTML 파일로 저장한다. react-dom createRoot, html2canvas, Blob 진행률 UI 표시
전체PPTX 전체 데이터로 네이티브 차트·표 중심의 여러 슬라이드 .pptx를 한 파일로 생성한다. pptxgenjs 캡처 파이프라인 없이 데이터 직접 반영
페이지 탭 · 이전/다음 · 도트 표지·차트 3P·데이터 4P, 총 8개 화면을 전환한다. React 상태. 차트 구간은 chart.js / react-chartjs-2 캡처 전 차트 렌더 대기 시간이 페이지별로 다름
목록에서 데이터를 가져올 때는 화면 로드 시 axios로 /api/pptData.json을 읽습니다. 위 표의 내려받기 관련 버튼은 그 결과 객체를 사용합니다.
